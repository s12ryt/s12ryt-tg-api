# 項目表

## 專案概況

- 主倉庫：`https://github.com/s12ryt/s12ryt-tg-api`
- 目前版本：`1.8.5`，日期見 `CHANGELOG.md`。
- 語言實作：Node.js 版為主要維護版本；Python 版已停止維護並改名為 `python(not-supported)/` 作歷史參考。
- 專案用途：Telegram Bot + Web Console + OpenAI/Anthropic-compatible API proxy，用一組本服務 API Key 管理多個上游 AI provider。
- 核心端點：`/v1/chat/completions`、`/v1/responses`、`/v1/messages`、`/v1/models`、`/health`。
- 支援 provider 類型：`openai_chat`、`openai_response`、`anthropic`、`google`。
- 授權：AGPL-3.0。

## 開發與執行

- Node 版本：`nodejs/package.json` engines 為 `>=22`；`.node-version` 與 `.nvmrc` 目前釘 `24`。
- Node 套件：ESM + TypeScript + strict mode；主 scripts 都透過 `node scripts/with-memory.mjs` 套用記憶體限制。
- 常用指令：在 `nodejs/` 下執行 `npm install`、`npm run dev`、`npm run build`、`npm test`。
- 必填環境變數：`BOT_TOKEN`、`ADMIN_ID`；常用可選：`API_PORT`、`DATABASE_PATH`、`DEFAULT_API_URL`、`NODEJS_PLUGIN_PATHS`、`memory`、Cloudflare/GitHub/NPM proxy/mirror 相關變數。
- 真實 `.env` 不應被代理讀取或提交；只讀 `.env.example`。
- `nodejs/start.js` 是容器/面板通用啟動器：自動切 cwd、設定 V8 heap、缺少 node_modules 時可安裝、優先跑 `dist/index.js`，必要時 build 或 tsx fallback。
- `nodejs/scripts/with-memory.mjs` 會讀 cwd 或 fallback `nodejs/.env`，套用 lowercase `memory` 或 `MAX_OLD_SPACE`。
- `nodejs/src/memory.ts` 允許整數或一位小數，低於 64 MB 視為無效。

## 重要目錄

- `nodejs/`：Node.js 主線版本。
- `nodejs/src/index.ts`：主入口；驗證 env、初始化 DB、建立 grammY bot、註冊 Bot handlers、載入/啟動 plugins、啟動 Express API、可選 Cloudflare Tunnel、處理 shutdown。
- `nodejs/src/config.ts`：dotenv/env parsing；`NODEJS_PLUGIN_PATHS` 支援 comma/semicolon/newline。
- `nodejs/src/db/database.ts`：sql.js SQLite 檔案 DB、schema/migrations/cache/write queue/backup import-export；插件 storage table 由 plugin services 建立。
- `nodejs/src/api/`：Express API proxy、auth/rate/quota middleware、usage/cost tracking、provider routing、format conversion、thinking effort 注入。
- `nodejs/src/api/providers/`：OpenAI Chat、OpenAI Responses、Anthropic Messages、Google Gemini adapters。
- `nodejs/src/bot/`：Telegram Bot filters/keyboards/handlers；user/admin/limit/update/web/backup handlers 是主要互動入口。
- `nodejs/src/web/`：Web Console 後端，`auth.ts` 為 OTP + session，`routes.ts` 為 `/web/api/*` REST routes。
- `nodejs/web/`：Vanilla JS SPA 靜態前端，深色主題，hash routing，無前端框架。
- `nodejs/src/plugins/`：Node.js plugin contract、manager、services facade。2026-07-05 優雅化重構後共 10 個檔案：`types.ts`（公開型別契約，`PluginContext`/`NodeJsPlugin`/`NodeJsPluginContext` 等，未變動僅補 JSDoc）、`index.ts`（唯一對外 barrel export，未變動僅補 JSDoc）、`manager.ts`（編排邏輯：validate/import/activate plugin、9 個 export 生命週期函式，簽名不變）、`services.ts`（`context.services` facade 組裝：auth/storage/events/scheduler/providers/db）、`pluginNaming.ts`（ID/檔名淨化驗證）、`pluginPathResolver.ts`（插件入口路徑解析，含測試依賴的英文 console.warn）、`pluginManifest.ts`（manifest 讀寫 + `InstalledPluginRecord`/`PluginInstallKind`/`PluginInstallInput` 型別）、`pluginRegistry.ts`（`PluginRegistry` class + singleton，封裝 manager.ts 全域可變狀態）、`pluginEventBus.ts`（`PluginEventBus` class + singleton，封裝事件監聽）、`pluginTimerRegistry.ts`（`PluginTimerRegistry` class + singleton，封裝計時器）。
- `plugin-example/`：Node.js-only 示範插件；獨立倉庫為 `https://github.com/s12ryt/s12ryt-nodejs-plugin-example`，主專案內仍保留範例 source/dist/README。
- `python(not-supported)/`：Python 歷史版本，FastAPI + python-telegram-bot + aiosqlite；已停止維護，除非使用者明確要求，後續不要優先修改。
- `scripts/vps.sh`：Linux VPS 互動部署腳本，支援 systemd 與 Docker。
- `agent/`：本地代理接手紀錄；被 `.gitignore` 忽略，更新後通常不會出現在 git tracked diff。

## 部署與 CI

- `Dockerfile`：multi-stage，build/runtime 都使用 `node:24-bookworm-slim`；build `npm ci --engine-strict && npm run build`；runtime 只裝 production deps，copy `dist/scripts/web/start.js/VERSION`，以 `node` user 執行並 expose 8000。
- `docker-compose.yml`：預設 image 為 `ghcr.io/s12ryt/s12ryt-tg-api:latest`，掛載 `./nodejs/data:/app/nodejs/data`，使用 `/health` healthcheck。
- Docker VPS 模式必須 pull GHCR image，不 clone、不在 VPS build；資料/設定目錄使用 `/opt/s12ryt-tg-api-docker`。
- systemd VPS 模式才 clone/update repo；預設 `/opt/s12ryt-tg-api`，service user `s12ryt`，Node 22.17.1。
- `.dockerignore` 排除 `.env*`、`agent`、`thoughts`、`node_modules`、`dist`、`nodejs/data`、Python 等。
- `.github/workflows/nodejs-ci.yml`：Node 22/24 matrix，`npm ci --engine-strict`、audit、build、test。
- `.github/workflows/ghcr.yml`：main/tag/path change 建置並推送 GHCR，包含 branch/tag/sha/latest metadata tags。
- `.github/workflows/release.yml`：main 更新 `latest` prerelease，semver tag 建 stable release。

## API 與權限重點

- API auth 支援 `Authorization: Bearer`、`x-api-key`、`x-goog-api-key`、`?key=`；API Key 必須是 `sk-s12ryt-` 前綴。
- Public skip：`/`、`/health`、`/docs`、OPTIONS、`/web`。
- `/web` static/router 在 API auth/rate/quota 前掛載，使用獨立 OTP/session auth。
- `/plugins` router 在 API auth/rate/quota 後掛載，因此 plugin HTTP routes 繼承 API Key 控制；Bot command plugin handler 需自行實作業務權限。
- rate limiter 以 user:key 做 RPM/TPM/concurrency；admin bypass；有效限制使用 60s cache。
- quota checker 檢查日/月 token 與 cost；0 表示 unlimited；admin bypass。
- provider API key 可為 JSON array 或 legacy single key；支援 `failover`、`round_robin`、`random`，連續失敗 3 次 suspend 60s。
- usage tracker 優先用 provider usage；fallback 為 OpenAI local tokenizer、Anthropic count_tokens、Google countTokens，再到 CJK-aware heuristic。
- thinking effort 支援 model suffix 與 request body 欄位，等級 `xhigh/high/medium/low/minimal/none`，跨 Chat/Responses/Messages/Coding fallback 保留。

## Web Console 重點

- OTP token 5 分鐘有效、一次性使用；session 24 小時有效，每 10 分鐘清理。
- Web auth 每次 request 重新檢查 admin 或 active user；disabled user session 會失效。
- User pages：dashboard、keys、usage、coding、limits、restrictions。
- Admin pages/routes：providers/prices/users/groups/keys/limits/restrictions/all-usage/system-usage/plugins/settings/update/restart/rollback/API logs/model mapping。
- Web plugin install 支援 upload、GitHub raw/repo/tree/blob，source size 上限 1 MiB，入口限 `.js/.mjs`。

## 插件系統重點

- `PluginContext` 包含 readonly `config`、bot、router、advanced app escape hatch、logger、`services`、`registerBotCommand`、`usePluginMiddleware`。
- `context.services` 僅提供安全 facade：auth、plugin-scoped storage、events、managed scheduler、providers read-only facade、db read-only facade。
- 不得暴露 raw DB、provider secrets、完整 API keys、任意核心寫入能力給插件。
- Storage 使用 `plugin_storage` namespaced JSON KV，plugin/key 名稱有限制，單值上限 256 KiB，目前不包含在主 `/backup` JSON。
- Scheduler managed timer 最小 1000ms，plugin stop 會清理 timer/event listener。
- `NODEJS_PLUGIN_PATHS` 可指 `.js/.mjs` 檔或目錄；目錄入口順序：`plugin.json` main、`package.json` module/main、`index.mjs`、`index.js`。
- 缺路徑、非 JS、無入口會 concise warning 並 skip，不應讓 dynamic import 直接炸掉主程式。

## 測試覆蓋

- 測試框架：Vitest + tsx/supertest；`tests/setup.ts` 設定預設 `BOT_TOKEN`、`ADMIN_ID`。
- 主要測試檔：`api_server.test.ts`、`auth.test.ts`、`config.test.ts`、`database.test.ts`、`pluginManager.test.ts`、`pluginServices.test.ts`、`providerAdapters.test.ts`、`quotaChecker.test.ts`、`rateLimiter.test.ts`、`responses.test.ts`、`security_coding.test.ts`、`thinkingParser.test.ts`、`updater.test.ts`、`web_routes.test.ts`、`anthropic_provider.test.ts`、`backupHandlers.test.ts`。
- 修改 shared API/DB/plugin/auth/rate/quota/thinking/web routes 時，至少跑對應 test；跨層改動優先跑 `npm run build` 與 `npm test`。
